1. What is the main value of Java Server Pages?
    • Using JSP, we avoid the need to write println with html tags. It also adds the flexiblity to change the presentation of the web page without the need to make changes to the other connected parts.
  2. How are JSP pages related to servlets?
    • Basically, when the container sees the JSP, it eventually converts it into servlet which itself explains the relationship between JSP and servlets.
  3. How are JSP pages related to HTML?
    • Any text outside the directives, scriptlets, or expressions are converted into HTML , HTML part of JSP is used by the browser to render content.
  4. Give an example of a JSP scriptlet and show how it will look in the JSP servlet.
    • JSP scriptlet: <%! public int add(int a, int b) { return a + b; } %>
    • JSP servlet: public int add(int a, int b) { return a + b; }
  5. Give an example of a JSP declaration and show how it will look in the JSP servlet.
    • <%! int i = 1; %>
    • int i = 1;
  6. Give an example of a JSP comment and show how it will look in the JSP servlet.
    • <%-- comment --%> this gets ignored in the JSP servlet

  8. Give an example of a JSP directive and show how it will look in the JSP servlet.
    • <%@ page import="java.util.Date" %>
    • import java.util.Date;
  9. Explain how an EL expression is evaluated.
    • First the container looks at the EL expression. Then it check the page scope. If it is found it will be it can be used for further purpose. Otherwise, the container keeps searching, until it founds it, all the way to application scope. The container ignores the expression if the search result is not positive.
  10. Explain how servlet attributes are involved in EL expressions.
    • Servlet attributes are involved in EL expression by defining the scope.
  11. Explain how servlets and JSPs use request dispatch to interact.
    • The request dispatch enables the client to send the same request and response throughout and therefore the JSP basically receives the values at the level of the request scope from the servlet seamlessly
