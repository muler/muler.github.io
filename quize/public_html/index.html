1. What is the difference between a web server and a web container?
    • Web server listens to requests from clients and provides a response. Web container,however, manages the components such as Servlets
  2. What is a servlet?
    • Servlet is a server side program which processes requests according to the requirements.
  3. How do web servers and web containers interact with servlets?
    • Web server receives a request, web container passes the request to the servlet after consulting the web descriptor
  4. Who creates request objects?
    • Servlet container
  5. What are the states in the servlet lifecycle?
    • First, servlet class is loaded followed by instanitiation. Then, the servlet calls service() method to process client's request before it is terminated and garbage collected by JVM garbage collector
  6. Who calls init and when?
    • By the servlet container and it is called once at the beginning of client's request
  7. Which of init, service, and doGet should you override?
    • service and doGet
  8. In what sense are servlets multi-threaded?
    • The servlet container creates multi-thread where each thread excutes the service() method of a signle instance of servlet
  9. What are the implications of this for servlet instance variables?
    • Poor implementation may lead to threadlocks
