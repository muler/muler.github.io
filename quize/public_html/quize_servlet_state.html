1. Can post requests be bookmarked? What are the problems?
    • No, they cannot mainly because all the form parameters will be lost
  2. What is the purpose of request dispatching?
    • It receives requests from the client and sends them to any resource on the same server.
  3. What is the difference between redirect and request dispatch?
    • In redirect, control can be redirect to resources to different servers or domains whereas using request dispatch, control can be forward to resources available within the server from where the call is made.
  4. What is an attribute ?
    • An attribute is one that enables servlets to share information among themselves
  5. What is the difference between attributes and parameters?
    • Both enables to share informations sharing among servlets. However, parameters are read only whereas the attributes could be created, added, or removed.
  6. What are dangers of using attributes?
    • It could lead to sharing sensitive information.
  7. What does it mean to say that http is stateless? Give an example of a stateful protocol.
    • Http is stateless in the sense that once the request processing the complete, the parameter values are lost. TCP protocol is stateful protocol.
  8. Give 5 different methods for maintaining state information (count each attribute scope as one method).
    • request object, session object, application object, cookies, hidden form
  9. How long does a session last?
    • For as long as the session timeout
  10. What is a cookie, and how long does a cookie last?
    • Cookie is a text file stored on the client. Session cookies last for as long as the browser is open and persistent cookies last for as long as the maximum age.
  11. What is the purpose of URL rewriting?
    • To alter the parameter values in the URL
  12. Why does the request attribute report ‘null’ for the maintaining state demo?
    • If the request attribute reports null, that means that the request processing is complete and any second attempt to retain the value leads to null.
